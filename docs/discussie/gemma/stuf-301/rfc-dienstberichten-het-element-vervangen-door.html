RFC: In dienstberichten het element <vrijeParameters> vervangen door <aanvullendeElementen>

In sectie 7.2.3 de volgende regel

Eventuele extra parameters kunnen worden meegegeven door het element <vrijeParameters> als laatste element op te nemen.

vervangen door

Eventuele extra gegevens kunnen worden meegegeven door het element <aanvullendeElementen> zoals gedefinieerd in sectie 3.2.4.2 als laatste element op te nemen.

Tijdens de expertgroep gaf ik al aan dat ik de oplossing te zwaar vond voor parameters. Het is mij toen duidelijk gemaakt/geworden dat het hier niet om parameters gaat, maar om extra gegevens bij de entiteit. Daarom zouden wij (Centric) willen voorstellen dat de tekst "Eventuele extra parameters kunnen worden meegegeven door het element <StUF:aanvullendeElementen> als laatste element op te nemen" wordt gewijzigd in "Eventuele extra gegevens kunnen worden meegegeven door het element <StUF:aanvullendeElementen> als laatste element op te nemen" (typografie niet meegenomen, uiteraard).

Hiermee wordt verdere spraakverwarring in de toekomst wellicht voorkomen.
