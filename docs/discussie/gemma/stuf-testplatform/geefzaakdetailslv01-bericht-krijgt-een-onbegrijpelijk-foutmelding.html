geefZaakdetails_Lv01 bericht krijgt een onbegrijpelijk foutmelding terug

Het bericht in de bijlage wordt in de testset uitvoering afgekeurd op code ZKDM000005 met de melding Vraag (zakLv01) en antwoord (zakLa01) berichten moeten aan regels in tabel 4.1.2.2 voldoen. 

De implementatie leert ons dat het volgende wordt gecontroleerd:
 

def list = ["//zkn:gelijk/zkn:identificatie",
        "//zkn:scope/zkn:object"]

return ZknUtil.checkRequiredElements(list, request)

beide elementen zijn aanwezig en volgens de Xsd en de Ad-Hoc test correct. Curieus feit is dat dit bericht in een vorige test geen meldingen opleverde.

Graag vernemen wij wat er aan scheelt.

Ik heb hetzelfde bericht via SOAPui ook zonder problemen kunnen aanbieden.

Na een debug sessie met Fiddler ben ik erachter gekomen dat het enige verschil is dat wij met SOAPui geformatteerde Xml hebben verstuurd. WCF stuurt ongeformatteerde Xml. Wanneer we met SOAPui hetzelfde bericht ook on-geformatteerd verzenden zien we de foutmelding terug.

Het probleem zit dus aan jullie kant in de lineparser en niet in de inhoud van het bericht.
