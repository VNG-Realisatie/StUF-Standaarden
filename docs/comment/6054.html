Voorgestelde mapping:
BAG-MUT Mutatie naar aanleiding van signalering voor mutaties als mutatiedatum actueel is geworden
BGR-COG Constatering nieuw object voor toevoegingen als mutatiedatum actueel is geworden

Pieter Dijkstra

Nog een aanvulling van mijn kant als oplossing voor ontbrekende objecten in StUF voor specifieke gebeurtenissen.

BGR-SSVSAMEN: Deze gebeurtenis kan zonder Pand worden doorgevoerd. Eventuele pandmutaties kunnen via een separate gebeurtenis (BGR-KVO). 
BGR-SSVSPLITS: Deze gebeurtenis kan zonder Pand worden doorgevoerd. Eventuele pandmutaties kunnen via een separate gebeurtenis BGR-KVO)
BGR-COG: OPR en WPL mappen via BRA-BOR en BRA-BWP (+ BAG-Mut)
BAG-AGO: Ontbrekende objecten (LIG, STA, OPR en WPL) mappen via BGR-ISLLP,  BGR-ISLSP, BRA-IOR en BRA-IWP (of BAG-MUT)
BAG-FGO: Voor ontbrekende objecten (LIG, STA, OPR en WPL) mappen via BAG-MUT
BGR-HMO: Voorgestelde mapping naar BAG-COR voldoet. Als alternatief is mapping via BAG-AGO (of  BGR-ISLLP,  BGR-ISLSP, BRA-IOR en BRA-IWP) mogelijk.

Volgens mij is het op deze manier mogelijk alle gebeurtenissen op een goede manier te mappen zonder wijziging in de xsd's.
